MutationObserver 接口

概述

MutationObserver接口提供了监视 DOM树变动 的能力。DOM 的任何变动，比如节点的增减、属性的变动、文本内容的变动，这个接口都可以得到通知。它被设计为旧的Mutation Events功能的替代品，该功能是DOM3 Events规范的一部分。

概念上，它很接近事件，可以理解为 DOM 发生变动就会触发 MutationObserver 事件。但是，它与事件有一个本质不同：事件是同步触发，也就是说，DOM 的变动立刻会触发相应的事件；MutationObserver 则是异步触发，DOM 的变动并不会马上触发MutationObserver ，而是要等到当前所有 DOM 操作都结束才触发，只触发一次。

这样设计是为了应付 DOM 频繁变动的特点。举例来说，如果文档中连续插入1000个<p>元素，就会连续触发1000个插入事件，执行每个事件的回调函数，这很可能造成浏览器的卡顿；而 MutationObserver 完全不同，只在1000个段落都插入结束后才会触发，而且只触发一次。

MutationObserver 有以下特点。

  • 它等待所有脚本任务完成后，才会运行（即异步触发方式）观察期的回调函数。
  • 它把 DOM 变动记录封装成一个数组进行处理，而不是处理一条条个别 DOM 变动记录。
  • 它既可以观察 DOM 的所有类型变动，也可以指定只观察某一类 DOM 变动。

观察器所能观察的 DOM 变动类型（即上面代码的options对象），有以下几种。

  • childList：子节点的变动（指新增，删除或者更改）。

想要观察哪一种变动类型，就在option对象中指定它的值为true。需要注意的是，必须同时指定childList、attributes和characterData中的一种或多种，若都不指定将报错。

其它属性 4个

除了变动类型，options对象还可以设定以下属性：

  • subtree：布尔值，表示是否将该观察器应用于该节点的所有后代节点。
  • attributeOldValue：布尔值，表示观察attributes变动时，是否需要记录变动前的属性值。
  • attributeFilter：数组，表示需要观察的特定属性（比如['class','src']）。
  • characterDataOldValue：布尔值，表示观察characterData变动时，是否需要记录变动前的值。

MutationRecord 对象

每个 MutationRecord 实例对象 代表一个独立的 DOM 变化。DOM 每次发生变化，就会生成一条变动记录（MutationRecord 实例）。该实例包含了与该次DOM变动相关的所有信息。MutationObserver 处理的就是一个个MutationRecord实例所组成的数组，该数组就是MutationObserver构造函数的回调函数的第一个参数。

MutationRecord对象包含了DOM的相关信息，有如下属性( 9个)

一条变动记录的属性 9个

  • type：如果是一个属性变化，则返回 “attributes”；如果是一个 characterData 节点的变化，则返回”characterData”；如果是子节点树变化，则返回 “childList”。
  • target：发生变动的DOM节点。
  • addedNodes：返回添加的节点。如果没有节点被添加，则该属性将是一个空的 NodeList
  • removedNodes：返回移除的节点。如果没有节点被移除，则该属性将是一个空的 NodeList
  • previousSibling：返回被添加或移除节点之前的兄弟节点，或 null。
  • nextSibling：返回被添加或移除节点之后的兄弟节点，或 null。
  • attributeName：返回被修改属性的属性名，或 null。如果设置了attributeFilter，则只返回预先指定的属性。
  • MutationRecord.attributeNamespace：返回一个字符串，表示被修改属性的命名空间，或 null。
  • oldValue：变动前的值。返回值取决于 MutationRecord.type，对于属性变化，返回变化之前的属性值；对于 characterData，返回变化之前的 data；对于子节点树变化，返回 null。

ParentNode 接口，ChildNode 接口

节点对象除了继承 Node 接口以外，还会继承其他接口。ParentNode接口包含了所有父节点共有的属性和方法，这些方法用于处理子节点，且多用于处理元素子节点。ChildNode接口包含所有子节点共有的方法，提供一些相关方法。ParentNode接口 和 ChildNode都是原始接口，并且不能创建此类型的对象。

  • ParentNode 接口
    • ParentNode.children
    • ParentNode.firstElementChild
    • ParentNode.lastElementChild
    • ParentNode.childElementCount
    • ParentNode.append()，ParentNode.prepend()
  • ChildNode 接口
    • ChildNode.remove()
    • ChildNode.before()，ChildNode.after()
    • ChildNode.replaceWith()

ParentNode 接口

如果当前节点是父节点，就会继承ParentNode接口。由于只有元素节点（element）、文档节点（document）和文档片段节点（documentFragment）可以作为父节点，拥有子节点，因此只有这三类节点会继承ParentNode接口。

ParentNode.childElementCount

childElementCount属性返回一个整数，表示当前节点的所有元素子节点的数目。如果不包含任何元素子节点，则返回0。

ParentNode.append()，ParentNode.prepend()

append方法为当前节点追加一个或多个子节点，位置是最后一个元素子节点的后面。

该方法不仅可以添加元素子节点，还可以添加文本子节点。

注意，该方法没有返回值。

prepend方法为当前节点追加一个或多个子节点，位置是第一个元素子节点的前面。它的用法与append方法完全一致，也是没有返回值。

注意区分，Node接口有 node.appendChild()方法，类似，还有node.insertBefore()。

ParentNode.querySelector()

​ 返回以当前元素为根元素，匹配给定选择器的第一个元素子节点

ParentNode.querySelectorAll()

​ 返回一个静态 NodeList 实例，表示以当前元素为根元素，匹配给定选择器组的元素列表

ChildNode 接口

如果一个节点有父节点，那么该节点就继承了ChildNode接口。它通过Element、DocumentType 和 CharacterData 对象实现。七种类型的节点中，只有 文档类型节点，元素节点，文本节点，注释节点 可以作为子节点。

注：

​ CharacterData 是一个抽象接口（abstract interface）代表节点对象包含的字符。抽象接口，意味着没有 CharacterData 类型的对象。 它是在其他接口中被实现的，如 Text、Comment 或 ProcessingInstruction 这些非抽象接口。

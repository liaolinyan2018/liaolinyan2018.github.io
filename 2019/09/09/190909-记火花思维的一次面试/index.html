then() 方法返回一个 Promise 。它最多需要有两个参数：Promise 的成功和失败情况的回调函数。

也就是说 then 方法 预期参数类型是 函数类型。

如果忽略针对某个状态的回调函数参数，或者提供非函数 (nonfunction) 参数，那么 then 方法将会丢失关于该状态的回调函数信息，但是并不会产生错误。如果调用 then 的 Promise 的状态（fulfillment 或 rejection）发生改变，但是 then 中并没有关于这种状态的回调函数，那么 then 将创建一个没有经过回调函数处理的新 Promise 对象，这个新 Promise 只是简单地接受调用这个 then 的原 Promise 的终态作为它的终态。

2.4 考察 try...catch

问输出结果：1 2 Error: 3

这道题我也答错了，我的答案是：1 undefined 2，原因是我没有 理解 try…catch。我理解成了 执行完 try 部分同步代码，就执行 catch 部分的 同步代码，因此我认为 先输出1，而抛出错误代码是异步的，因此 e 是 undefined，同步代码执行完了，等异步代码 setTimeout 执行时，抛出的错误就不会被捕获了。现在感觉我的思维号好反人类哦。

正解：

MDN

try语句包含了由一个或者多个语句(可以是同步代码、异步代码)组成的try块, 和至少一个catch子句或者一个finally子句的其中一个，或者两个兼有， 下面是三种形式的try声明：

  1. try...catch
  2. try...finally
  3. try...catch...finally

try...catch语句中，catch子句包含try块中抛出异常时要执行的语句。也就是，你想让try语句中的内容成功， 如果没成功，这时你可以在catch语句中控制接下来发生的事情。 如果在try块中有任何一个语句（或者从try块中调用的函数）抛出异常，（就不再执行该语句后面的代码），控制立即转向catch子句。如果在try块中没有异常抛出，会跳过catch子句。

finally子句在try块和catch块之后执行但是在下一个try声明之前执行。无论是否有异常抛出或捕获，它总是执行。

可以嵌套一个或者更多的try语句。如果内部的try语句没有catch子句，那么内部的try语句抛出的异常将会进入包裹它的try语句的catch子句。

可以用try语句去处理 JavaScript 异常

也就是说，catch 子句时刻处于在监听 try语句块是否抛出异常。 只有捕获到异常时才会进入catch子句。这样上面的面试题答案就容易知道了。

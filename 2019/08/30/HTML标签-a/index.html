a标签

a是anchor锚的缩写， HTML<a>元素(或锚元素)可以创建一个到其他网页、文件、同一页面内的位置、电子邮件地址或任何其他URL的超链接。

URL 相关属性

<a>元素有一系列 URL 相关属性，可以用来操作链接地址。这些属性的含义，可以参见Location对象的实例属性。

  • href：完整的 URL
  • protocol：协议（包含尾部的冒号:）
  • username：主机名前的用户名
  • password：主机名前的密码
  • host：主机和端口（默认端口80和443会省略）
  • hostname：主机名
  • port：端口
  • origin：协议、域名和端口
  • pathname：路径（以/开头）
  • search：查询字符串（以?开头）
  • hash：片段识别符（以#开头）

上面的主机 就是 域名。

除了origin属性是只读的，上面这些属性都是可读写的。上面这些属性就是把完整的 url 拆分组装。

hreflang 属性

hreflang属性用来读写<a>元素的 HTML 属性hreflang，表示链接指向的资源的语言，比如hreflang="en"。

referrerPolicy 属性

referrerPolicy属性用来读写<a>元素的 HTML 属性referrerPolicy，指定当用户点击链接时，如何发送 HTTP 请求头的referer字段。

HTTP 头信息的referer字段，表示当前请求是从哪里来的。它的格式可以由<a>元素的referrerPolicy属性指定，共有三个值可以选择。

  • no-referrer：不发送referer字段。
  • origin：referer字段的值是<a>元素的origin属性，即协议 + 主机名 + 端口。
  • unsafe-url：referer字段的值是origin属性再加上路径，但不包含#片段。这种格式提供的信息最详细，可能存在信息泄漏的风险。

rel 属性

rel属性用来读写<a>元素的 HTML 属性rel，表示链接与当前文档的关系。

tabIndex 属性

tabIndex属性的值是一个整数，用来读写当前<a>元素在文档里面的 使用Tab 键遍历的顺序。该属性是一个全局属性。

target 属性

target属性用来读写<a>元素的 HTML 属性target。该属性有4个值。该属性指定在何处显示链接的资源

方法

<a>元素的方法都是继承的，主要有以下三个。

  • blur()：从当前元素移除键盘焦点，详见HTMLElement接口的介绍。
  • focus()：当前元素得到键盘焦点，详见HTMLElement接口的介绍。
  • toString()：返回当前<a>元素的 HTML 属性href。

任务四错题汇总

发表于 2018-10-28

在命令行里书写命令时，大小写无所谓，比如 cd .. 和 CD .. 没区别。×
命令行如何进入上级目录。答：[cd ..] 或[cd ../]
命令行如何进入当前目录中的 a 目录。答：[cd a] [cd a/] [cd ./a] [cd ./a/]
目前我在 /a/b 目录，输入 cd /c/ 我会进入哪个目录？答： /c/
命令行如何在当前目录创建一个目录 xxx？答： [mkdir xxx] [mkdir ./xxx] [mkdir -p “xxx”]
目前我在 /a/，我想创建 /a/b/c/d/e/ 目录，用一个 mkdir 命令如何实现？答：[mkdir -p /a/b/c/d/e/] [mkdir -p ./b/c/d/e/]
如何删除目录 /a/xxx/ ?答：rm -rf /a/xxx/
目前我在 /tmp/ 目录，运行 git clone git@github.com:FrankFang/nodejs-test.git，请问 /tmp/ 目录里会多出一个什么目录？ 答：[nodejs-test]
将 xxx 文件变动加入 git 暂存区的命令是什么？ 答：git add xxx
将 xxx 文件变动提交到 git 本地仓库的命令是什么？ 答:git commit xxx
git push 之前最好 git pull 一下? 答：√
git 的默认远程仓库名叫做什么？ 答：origin
git 的默认分支叫做什么？ 答：master
git课外资料： http://www.runoob.com/git/git-tutorial.html

快速搭建博客第二篇

发表于 2018-10-28

第二篇文章还是关于用Hexo和github如何快速搭建一篇博客，总结一下：
搭建博客四步骤：
hexo new “博客文章名” //hexo会帮我们在..\hexo\source_posts下生成 博客文章名.md 文件
start 博客路径 //注意博客路径要写完整,上一步有提示。用VScode打开这个博客文件，编辑博客内容
//部署到github之前可以打开站点配置文件_config.yml进行修改配置，不熟悉最好google找篇教程）
hexo clean //以下三步是部署到github
hexo g //全称hexo generate,执行之后,hexo就会在public文件夹生成相关html文件，这些文件将来都是要提交到github上 // 一个叫 [你的用户名.github.io]的仓库上去的 .hexo g 生成静态文件
hexo d //全称hexo deploy,执行之后,就部署到github了。

文档流是什么？

文档内元素的流动方向。每个内联元素（inline）从左到右流动，每个块级元素（block）从上到下流动，每个块级元素独占一行。给元素加边框border是很好的调试方法。

说一下自己的感受，调试了一点CSS样式，工具很好用，就是调试需要耐心，往往根据浏览器显示的为准，有点玄学的感觉，所以往往需要选取容错率较高的CSS样式实现方法。CSS样式是一环套一环的，“牵一发而动全身”，可见布局很重要，CSS经验也很重要。

高度是由什么决定的

  1. 内联元素的高度

    内联元素的高度等于行高。

    内联元素的宽度(width)和高度（height）不能由程序员设置，会随里面内容的宽度和高度自动改变。但是你可以把内联元素设置为块级元素其中内联元素有个奇特的地方，当一个内联是一个很长的单词的时，页面就算不够大容纳不下这个单词，默认情况下这个单词是不会分开，浏览器会把这个单词看作一个整体，英文的不可分割性原则，但是中文汉字就算是一个词语也会分开。但是可以通过设置word-break：break-all;样式强行分开。内联元素是inline

    内联元素的高度是不可测的，当font-size较小时，可以通过设置line-height设置,辅助以加padding。比如span的高度确定就是个复杂的问题，具体以浏览器显示为主。

  2. 块级元素的高度

    块级标签元素的高度：由其内部文档流元素 的高度总和 决定【背】也就是说，块级标签内部没有元素的时候 高度为0 height=0






  3. 内联块级元素

    display:inline-block;，方说这个是这个是历史上最大的错误，不知道从何说起？这个内联块级元素把块级元素变成内联元素以后，把独占一行的块级元素排成一行，就像导航栏的浮动效果。方说尽量用浮动，即子元素标签加float，父元素标签加类.clearfix,再给父元素加个固定样式，消除float的bug。内联块级元素不会独占一行，而且宽度（width）和高度(height)可以由程序员设置，拥有内联和块级元素的特点

box 是啥（盒模型）

box-sizing有两个值分别为content-box和border-box，区别是：

content-box 的内容的width 不包括 padding 和 border。故其实际算出来的 盒子的高度和宽度 >= 内容的width

border-box 的 width 包括 padding 和 border，故其实际算出来的 盒子的高度和宽度 就是（=） 内容的width。

宽度是有什么决定的

块级元素宽度是自适应的，内联元素的宽度不可测。

position 的 5 个取值

① position=fixed;相对窗口定位。脱离文档流第一种。
② 祖宗（最近的父元素）样式加position=relative;孙子（儿子/小辈）样式加position=absolute;则小辈相对于有relative的最近一个祖宗绝对定位。脱离文档流第二种。
③ position:sticky;
④ position:static;

今天学几个比较难的标签

学习标签就是重点学标签的语义和属性。标签单词懂了，语义一般就懂了。而标签有什么属性则是需要查文档，然后练习和记忆的，标签常见的属性是必须熟悉熟悉熟悉的。

2. a标签



a是anchor锚的缩写， HTML<a>元素(或锚元素)可以创建一个到其他网页、文件、同一页面内的位置、电子邮件地址或任何其他URL的超链接。介绍<a>的三个常用属性target href download。



target属性有四个值，分别是单词对应意思。

<a href="http://qq.com" target="_blank">blank_qq</a> <!--在新开空窗口打开-->

<a href="http://qq.com" target="_self">self_qq</a>  <!--在自己这个窗口（2.html）打开-->

<a href="http://qq.com" target="_parent">parent_qq</a> <!--在父亲窗口(这里是index.html)打开和iframe一起使用才有体会-->

<a href="http://qq.com" target="_top">top_qq</a> <!--在顶级窗口打开，在至少有祖孙三代才能体会到-->

效果：


点击第1个qq以后会新开窗口(即不会把爷爷index.html页面覆盖掉)打开


点击第2个qq以后在自己的窗口(3.html，不会覆盖掉父亲2.html页面)打开


点击第3个qq以后在父窗口(即会把父亲2.html页面覆盖掉)打开


点击第4个qq在顶层口打开(即覆盖掉爷爷index.html页面)，可以依次打开其他qq可深刻体会到效果

3. form标签



get 是获取这个页面内容，post 是上传数据到服务器。除了form，HTML没有其他标签能上传内容到服务器。a 标签是get，form 是post。get会默认把参数放到查询参数里面，直接出现在地址栏里，post会默认把参数放到 请求的第四部分里面。我们没有办法让get请求有第四部分。总结就是a标签和form标签都是发起一个请求的，只不过a标签是get一个页面，form标签是post一个页面。

form标签属性:action属性指定请求路径， method属性指定请求动词，值是post， target属性值和a标签的target属性使用方法相同。

<form action="users" method="post" target="与a标签一样用法">
 <input type="text" name="username">
 <input type="password" name="password">
 <input type="submit" value="提交"> <!--只有有了提交按钮才能将数据提交到服务器,submit是唯一能确定这个表单能提交的-->
</form>


4. input/button 标签



input最难记的是type属性值，有很多。input按type值确定功能。input一般是form标签的子元素。

<form action="users" method="post" target="与a标签一样用法">
 <input type="text" name="username">
 <input type="password" name="password">
 <input type="submit" value="提交"> <!--提交按钮-->
 <input type="button" value="button"> <!--这只是普通按钮-->
</form>

提交按钮和普通button按钮长得一样，但是功能不一样。



有个特例，当form表单里只有botton按钮时，botton会自动上升为submit按钮。如下的button就是submit按钮。
例1

<form action="users" method="post" target="与a标签一样用法">
<input type="text" name="username">
<input type="password" name="password">
<button>提交</button> <!--普通按钮升级成提交按钮-->
</form>


label和input配对使用作为form的子元素，label包围input，当用户在文字上点击就能选中复选框或者输入框
例2

<form action="users" method="post" target="与a标签一样用法">
<label>用户名<input type="text" name="username"></label>
<label>密码<input type="password" name="password"></label>
<label><input type="checkbox">苹果</label>
<input type="button" value="button">   
</form>

提交时会把参数都带上给服务器。

input其他的type值要多靠自己去试出来，然后做笔记。
input的属性值：type name id value

button前面讲了一点，注意input和button的区别，input没有子元素，button有子元素。form 标签里面的input若不加 name属性，那么在表单提交时，input 的值就不会出现在请求里，必须要有name属性才能提交数据到服务器，切记

5. table标签

table标签不常用。 HTML规定table标签只能有三个子元素thead、tbody、tfoot。子元素里面又可以有很多元素。

例一：

<table>
    <thead>
        <tr>
            <th></th><th>姓名</th><th>班级</th><th>分数</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th></th><td>小明</td><td>1班</td><td>96</td>
        </tr>
        <tr>
            <th></th><td>小红</td><td>2班</td><td>94</td>
        </tr>
        <tr>
            <th>平均分</th><td></td><td></td><td>95</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <th>总分</th><td></td><td></td><td>190</td>
        </tr>
    </tfoot>
</table>




例2：table的完整用法

 <table border=1>
    <colgroup>
        <col width=100>
        <col width=200 bgcolor=red>
        <col width=100>
        <col width=70>
    </colgroup>            <!--colgroup和col列配套使用-->
    <thead>
        <tr>
            <th></th><th>姓名</th><th>班级</th><th>分数</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th></th><td>小明</td><td>1班</td><td>96</td>
        </tr>
        <tr>
            <th></th><td>小红</td><td>2班</td><td>94</td>
        </tr>
        <tr>
            <th>平均分</th><td></td><td></td><td>95</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <th>总分</th><td></td><td></td><td>190</td>
        </tr>
    </tfoot>
</table>




注意当tbody、tfoot、thead、colgroup顺序混乱时，浏览器会自动更正他们的显示顺序依然是thead、tbody、tfoot、colgroup。当你thead、tbody、tfoot这些标签都不写的时候，浏览器会认为你写的都是tbody的内容，此时就会按你写的内容顺序展示给用户。当你在头head标签加入css样式可以去掉table之间的间隙，如下：

<head>
......
   <style>
     table{
        border-collapse: collapse;
        }
   </style>
</head>

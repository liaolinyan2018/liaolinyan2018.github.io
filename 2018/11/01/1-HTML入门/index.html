1-HTML入门

发表于 2018-11-01

关于HTML

HTML（超文本标记语言——HyperText Markup Language）是构成 Web 世界的一砖一瓦。它定义了网页内容的含义和结构。除 HTML 以外的其它技术则通常用来描述一个网页的表现与展示效果（如 CSS），或网页的功能与行为（如 JavaScript）

关于W3C

W3C是英文 World Wide Web Consortium 的缩写，中文意思是W3C理事会或万维网联盟。W3C组织是制定万维网标准的一个非赢利组织，像HTML、XHTML、CSS、XML的标准就是由W3C来定制。

关于MDN

MDN Web Docs（旧称Mozilla Developer Network、Mozilla Developer Center，简称MDN）是一个汇集众多Mozilla基金会产品和网上技术开发文档的免费网站。

简单说，W3C和MDN是平时学习前端的工具，我们可以到上去查找学习资料，比如要想知道一个标签的语义及用法就可以搜 标签 MDN/W3C。

学习HTML5

直接学习HTML5，HTML5容错率很高，兼容XHTML和HTML4.01。HTML5.1刚出来，还没开始使用。

HTML5跟H5没关系，H5就是能运行在微信上的网页。 H5可以用HTML5做，也可以用HTML4做，H5跟技术没有关系。

HTML5 文档

google搜 HTML spec 可以搜到 HTML5 的全英文档，看起来很痛苦因为是全英文的。不过大致浏览一遍，基本能把前端的单词都过一遍了。

在HTML5标签规范里规定<head></head><body></body>可以省略（omit），浏览器会自动帮你补齐。W3C 有个HTML 验证器，可以验证，检查错误。所以有了规范文档以后我们就知道哪些能写哪些不能写，可以说文档就是把html的语法总结到一个文档里。

语义化

语义化大概意思就是 你看到这个标签单词的意思就知道这段代码是什么意思了。使用有含义的标签。

<div>和<span>是没有什么特殊意义的标签，所以必须加上class。

<video>是加视频，<audio>是加音频。<svg>是不规则标签。kbd是键盘标签，<nav> 导航栏标签。当我们要看一个标签怎么用，一个办法看规范文档，但是文档都是英文的，难，第二个是看MDN，例如搜 strong MDN,自己写博客就可以参考MDN，了解一下MDN的历史，它也是一个文档的集合。遇到哪个什么标签用MDN就可以了。如何看MDN文档也值得学习。记笔记很重要，你面试会忘记的。

HTML 可替换元素 & 不可替换元素

HTML元素可分为 可替换元素 和 不可替换元素。可替换元素 往往没有实际的内容，即是一个空元素，通过浏览器根据元素的标签和属性，来决定元素的具体显示内容。

例如浏览器会根据<img>标签的src属性的值来读取图片信息并显示出来，而如果查看(x)html代码，则看不到图片的实际内容；又例如根据<input>标签的type属性来决定是显示输入框，还是单选按钮等。<img>和<input>都是可替换元素，没有实际的内容。

大部分HTML元素 是 不可替换元素，不可替换元素 是 元素内容直接表现给用户端。

例如：<p>段落的内容</p>
段落<p>标签是一个不可替换元素，它的内容文字 “段落的内容” 全被显示。

空标签

空标签 就是 无子元素的标签。例如： <img> <input> <meta> <link> <br>

查看所有空标签google： 空标签 mdn

仿页面思路

当我们要仿一个页面的时候，我们只用去想，这个内容是 标题、段落、图片、链接、导航栏、无序链表、描述链表、表格还是其他之类的，当我们实在想不出标签才用div，span标签，写html是一个很简单的过程，注意写的过程HTML的结构化，代码可读性强。

写 在线简历网站的 HTML

html只管内容，定义这是什么东西，不管样式，在HTML中没有内联元素和块级元素的区别，我跟你说这是个段落，不能告诉你这是什么样式的段落。CSS才是管样式。HTML是不管样式的，所以我们在写HTML标签的时候，一定不能管样式。一个标签是块级元素还是内联元素，要么是CSS默认的，要么是CSS设置的。

分析你看到的这个页面，由哪些部分（标签组成），写html的时候不用考虑样式切记。比如这个简历，上半拉由 图片logo<img>，导航栏<nav>组成，图片的地址暂时不知道用#代替，导航栏中的若干词条其实又是链接（内外跳转），用<a>分别包含各个词条，各个词条之间的关系又是 无序链表 ，所以用<ul><li>（unorderred list）将整个词条包起来。这就是上半部分的语义化。你看代码就能看出表达什么。通过这个在线的简历，我们发现<img> 和 导航栏 又是被划分 到一块了，所以用<div>将两个整体标起来，而<div>标签往往用一个class属性值对来区分,因为我们知道div标签是没有语义的，所以必须用class来标记它是属于哪一块，同时这样你的代码质量也更高（以后我们知道这是为了添加样式、定位更方便）。黑色banner意为广告横幅，在html里面就是一个 背景图片，而背景图片不用<img>标签，单独用<div>标签划分一个块。接着 我们看到页面最中间的位置，是我们简历最主要的部分，在html中最主要的部分用<main>标签框起来。然后在<main>标签里面写代码,首先我们要分析这个主要的部分里面的结构，是上下还是左右，我们的例子里是上下结构，上 又是一个左右结构，上左是一个图片，上右 是问候语，名字，职称，链表。我们用class命名的时候，命名的规范是告诉它是什么，而不是长什么样。用<div>分上下块，分别在上<div>块中再使用<div>分左右块，左 picture，右 text。左边同上，右边 问候语是没有专门的标签，用 <span> 包起问候语。
